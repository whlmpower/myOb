## 📖 文案内容


```markdown

刚工作，领导骂我"迟早要滚蛋"，后来我拿了6次最高绩效。

刚入职换了新主管，负责冷门业务，领导说我不汇报，没动静，明显落后于同期校招生。

校招SP没动静？那我先从搞动静做起。  
我学会了一招，来了项目绝不埋头苦干，先三句话讲清楚：任务内容、deadline、交付结果。关键时间点主动汇报，单独写项目周报。

汇报有啥难的？脸皮厚就行。正当我这么想，又被骂了。  
"你要有自己的思考，不是等别人给方案。"

那我就带着问题提出方案。  
以前：领导，我解决不了，跟主管要方案。  
后来：领导，我的想法是A或者B，缺点是，优点是，你帮我拍板。  
思考和方案是下属的事，拍板才是领导的事。

干了一段时间，同期校招生都做项目经理了，我还在默默干活，年底差绩效，没准还得我来背。

正好隔壁组要人支援，同事说糊弄糊弄就行。我干完活之后，顺手写了篇技术实操指南。主管的领导都点赞了，开大会特意说了我们组支援到位。  
别人不愿干的事，干了就是你的机会。

后来，绩效翻盘了，领导换了个人似的对我。 
这是阿里到国企的第16期，是我真实的经历，下期告诉你，真正的卷王从来不卷工作时长，我"卷"了连续两年最好绩效。

评论区聊聊：你第一次拿差绩效是什么感受？

```

### 第一部分：引导式评论（小号，发布后1分钟内发）

**目标**：提供讨论话题，引导用户表达观点

| 方案        | 评论内容                  | 适用场景         |
| --------- | --------------------- | ------------ |
| **A（推荐）** | 你们第一次拿差绩效是什么感受？       | 情感共鸣，开放性强    |
| **B（备选）** | 在大厂遇到过什么奇葩领导？         | 话题延伸，适合吐槽    |
| **C（深度）** | 楼主说的第三层"为领导争光"具体怎么操作？ | 干货追问，筛选高价值用户 |

**建议**：发A或B，二选一即可，不要全发。

---

### 第二部分：置顶评论（发布后3分钟内置顶）

**目标**：引导关注，预告下期，强化系列记忆


### 第三部分：自问自答评论（发布后10分钟内发）

**目标**：解答潜在疑问，提供额外价值

**问题型**（小号发问 + 大号回复）：

| 小号提问                     | 大号回复                                          |
| ------------------------ | --------------------------------------------- |
| **怎么为领导争光，这个实操起来很看机会啊"** | 除执行以外，要换位到领导视角想想，这个工作怎么能提升团队或领导自己的影响力，让他脸上有光。 |

**关键词**：「361制度」「大厂绩效」——引导用户搜索，提升搜索流量。

---

### 第四部分：用户评论回复模板

**目标**：让评论用户感到被重视，提升互动率权重

|用户评论类型|回复模板|
|---|---|
|共鸣型（"我也拿过3.5"）|握爪，当时感觉天都塌了，后来才懂是游戏规则没搞清|
|提问型（"第三层具体怎么操作？"）|简单说就是：活干了还要让领导知道。详细的我整理了一篇，私信「361」发你|
|吐槽型（"领导傻逼"）|理解，我当时也这么觉得，后来发现得先搞懂规则再谈别的|
|疑问型（"什么是361？"）|就是强制排名30%优秀、60%合格、10%待改进，私信「361」我发详细解读|
