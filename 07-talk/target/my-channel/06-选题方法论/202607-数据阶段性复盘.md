## 三、选题方法论：3维漏斗

### 第一维：方向锚定（必须满足）

所有内容必须落在三个系列之一：

| 系列          | 核心问题               | 判断标准                 |
| ----------- | ------------------ | -------------------- |

| **普通人实验室**  | 我观察/实验了一件事，得出什么感受？ | 有没有「我试了XX/我发现XX」的实验感 |

## 三、旧程序和普通人实验室本质区别

|**旧程序**|**普通人实验室**|
|---|---|---|
|**核心动作**|自我拆解|实验/观察|
|**触发方式**|一件小事 → 挖到原生家庭 → 拆心理模式|一件事/一个实验 → 记录过程 → 得出感受|
|**结构**|小事 → 原生家庭 → 心理模式 → 自我拷问|实验/观察 → 过程记录 → 结论感受|
|**情绪底色**|沉重、拷问、自我解剖|轻松、记录、恍然大悟|
|**你的例子**|房东保时捷 → 我爸说的「读书改变命运」→ 我到底在拼什么？|团购房便宜30万 → 但想走走不了 → 稳定是牢笼|
|**爆款潜力**|情绪浓度高，容易出金句，完播率高|话题普适性强，容易引发「我也这样」的共鸣|


**快速判断3问**：
1. 有没有原生家庭/童年经历？→ 有 = 旧程序
2. 有没有「我试了XX/我发现XX」的实验感？→ 有 = 普通人实验室
3. 有没有两边对比？→ 有 = 阿里→国企对比系列

---

### 第二维：话题价值判断（满足2项以上才做）

| 维度       | 高价值特征（+20分）                  | 低价值特征（0分）            |
| -------- | ---------------------------- | -------------------- |
| **搜索价值** | 有明确高搜索热词：阿里P7薪资、大厂裁员赔偿、应届生身份 | 没有明确搜索热词：「稳定工作真的快乐吗」 |
| **数字锚点** | 有具体可感知的数字：40万、4万、2小时、3个部门    | 只有抽象描述：「国企效率很低」      |
| **情绪锚点** | 命中年轻人集体情绪：裁员焦虑、薪资焦虑、职业选择焦虑   | 只讲个人感受               |
| **反差冲突** | 有强烈对比反差：大厂vs国企、高薪vs低薪、努力vs躺平 | 没有冲突，平铺直叙            |
| **独特性**  | 只有你能讲：exact的薪资数字、exact的裁员经历  | 随便一个从大厂到国企的人都能讲      |

**打分规则**：
- ≥80分：立即做
- 60-79分：可以做，但优化后再做
- <60分：直接pass

### 第三维：内容差异化设计（必须做）

三种方法：
1. **加exact经历**：别人讲「大厂vs国企薪资差距」，你讲「我阿里P7月薪4万，国企第一个月到手4700」
2. **加unique视角**：别人讲「应届生选大厂还是国企」，你讲「我师弟放弃40万选15万，我以为他疯了，直到我也被裁员」
3. **加personal情绪**：别人讲「国企稳定是牢笼」，你讲「我买了团购房便宜了30万，但现在想走也走不了，离职要赔20万违约金」

## 五、文案产出SOP：把你的文案产出效率提高3倍，同时保证爆款率

### 5.2 核心原则（来自dbs-content）

**原则1：文字洁癖是底线**——用自己的话，不用AI模板语言

**原则2：自媒体的本质是精神控制**——封面和标题是认知劫持，必须触发神经机制

**原则3：内容好坏 = 投入精力 × 对内容有正确理解**——不是写得多就是好，是写得准才是好

**原则4：先有产品后有内容**——文案必须服务于三个系列定位，不是为了自嗨

**原则5：知识博主的核心工作只有两个**——把事情搞清楚，把事情说清楚

### 5.3 5型钩子公式（前3秒决定生死）

前3秒用户问的不是「这个人说得对不对」，而是「这跟我有什么关系」。

前3秒必须是具体事件/画面，不是抽象道理。

| 钩子类型 | 适用场景 | 公式 | 你的例子 |
|----------|----------|------|----------|
| **数字冲击型** | 薪资、收入、赔偿类 | 具体数字+反差 | 「4万，这是在阿里的月薪，4700，这是国企第一个月的工资」 |
| **场景代入型** | 对比类、自我拆解类 | 具体场景+情绪 | 「在阿里，我最怕的就是等。到了国企，第一天学会的，就是等」 |
| **反转悬念型** | 普通人实验室、热点类 | 反常识+悬念 | 「师弟放弃40万大厂选15万国企，我以为他疯了，直到我也被裁员」 |
| **引用对话型** | 自我拆解、观点类 | 别人的话+你的反应 | 「被裁员那天，HR说：「可惜了，你真的很适合阿里。」」 |
| **设问引入型** | 热点类、福利类 | 问题+悬念 | 「堂堂全国大央企，你猜刚上线的报销系统到底有多难用？」 |

**钩子必须满足三个条件**：
1. 前3秒必须出现具体数字/具体场景，不能讲抽象道理
2. 必须有反差/冲突，让用户想知道「后来呢」
3. 必须和用户的痛点挂钩，让用户觉得「这跟我有关」

### 5.4 快-慢-停-快-慢模板（60-70秒口播专用）

```
【快铺垫（5秒）】：用钩子引入，快速交代背景，不要铺垫太多
【慢冲突（30秒）】：展开核心冲突，用具体场景、具体数字、具体对话，不要讲抽象道理
【停顿落金句（1-2秒）】：冲突到顶的时候，停1-2秒，抛金句，让用户截图
【快推进（20秒）】：快速推进到结论，不要拖泥带水
【慢收尾（5秒）】：引导互动/关注，用「你也有过这种经历吗？」「关注我，下期讲XXX」
```

**金句设计公式**：反差+共鸣+可截图
- **反差点**：和常识相反，或者和前面铺垫的内容相反
- **共鸣点**：说出用户想说但说不出来的话
- **可截图**：短、有冲击力、适合发朋友圈/抖音
