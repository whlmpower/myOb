# Omnisearch 使用指南

## 一、简介

Omnisearch 是 Obsidian 的增强搜索插件，比原生搜索更快、更智能，支持模糊匹配、中文分词、PDF 和图片 OCR 搜索。

---

## 二、基本使用

### 打开搜索

```
Ctrl/Cmd + O
```

这是 Omnisearch 的专属快捷键，按下后弹出搜索框。

或者：

```
Ctrl/Cmd + P → 输入 "Omnisearch" → 选择 "Omnisearch: Vault Search"
```

---

## 三、搜索方式

### 1. 关键词搜索
直接输入关键词：
```
项目管理
```
→ 列出所有包含"项目管理"的笔记

### 2. 多关键词搜索
用空格分隔多个关键词：
```
项目 进度 2025
```
→ 列出同时包含这三个词的笔记

### 3. 精确匹配
用英文双引号包裹：
```
"项目管理方法"
```
→ 只匹配完整出现"项目管理方法"的笔记

### 4. 排除关键词
用减号 `-` 排除：
```
项目 -归档
```
→ 包含"项目"但不含"归档"的笔记

### 5. 搜索标签
```
#读书
```
→ 找出所有包含 `#读书` 标签的笔记

---

## 四、搜索范围

Omnisearch 会搜索以下内容：

- ✅ 笔记标题
- ✅ 笔记正文内容
- ✅ 标签（#tag）
- ✅ Wiki 链接文字
- ✅ PDF 文件内容（需开启设置）
- ✅ 图片中的文字 OCR（需开启设置）

---

## 五、实用设置

> 设置 → Community plugins → Omnisearch

| 设置项 | 推荐 | 说明 |
|--------|------|------|
| **Ignore diacritics** | ✅ | 忽略音调符号 |
| **Images OCR** | ✅ | 搜索图片中的文字 |
| **PDF content** | ✅ | 搜索 PDF 内容 |
| **Weight folders** | 根据需要 | 给某些文件夹更高权重 |

---

## 六、与 Obsidian 原生搜索的区别

| 特性 | Obsidian 原生搜索 | Omnisearch |
|------|------------------|------------|
| 快捷键 | `Ctrl/Cmd + Shift + F` | `Ctrl/Cmd + O` |
| 搜索速度 | 一般 | 更快（模糊匹配） |
| 中文支持 | 一般 | 更好 |
| 搜索结果排序 | 按文件名 | 按相关性排序 |
| PDF/图片搜索 | ❌ | ✅ |
| 模糊匹配 | ❌ | ✅（打错字也能找到） |

---

## 七、实用技巧

### 快速打开笔记
```
Ctrl/Cmd + O → 输入笔记名 → 回车
```
比文件树找文件快很多！

### 模糊搜索
Omnisearch 支持模糊匹配，打错字也能找到：
```
项目管里
```
→ 仍然能找到"项目管理"相关笔记

### 搜索特定文件夹
在搜索结果中，每条结果会显示文件路径，可以快速识别所在文件夹。

---

## 八、快捷键自定义

> 设置 → 快捷键 → 搜索 "Omnisearch"

可以自定义：
- **Vault search** — 全库搜索
- **In-file search** — 当前文件内搜索
- **Vault search with backlinks** — 搜索包含反向链接的结果

---

## 💡 日常使用场景

```
场景1: 快速打开笔记
  Ctrl/Cmd + O → 输入笔记名 → 回车

场景2: 找某个主题的所有笔记
  Ctrl/Cmd + O → 输入关键词 → 浏览结果

场景3: 找昨天写的某个内容
  Ctrl/Cmd + O → 输入关键词 → 看日期排序

场景4: 找某个标签下的笔记
  Ctrl/Cmd + O → #标签名
```

---

## 💡 推荐

Omnisearch 是替代 Obsidian 原生搜索的最佳选择，日常用 `Ctrl/Cmd + O` 打开搜索，比在文件树里翻找高效很多。
